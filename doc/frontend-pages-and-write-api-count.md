# 前端页面与新增/修改接口梳理

> 统计依据：`doc/短信触达平台 V1.0 产品需求文档.md`、`doc/api-interface-spec.md`、当前 `apps/web/src/main.tsx`。
>
> 统计口径：
>
> - 前端页面按后台路由页面统计；弹窗、详情、表单作为独立交互面单独列出。
> - 新增/修改接口只统计 `POST` 以及会触发业务动作的接口；接口设计统一使用 `GET` 查询、`POST` 新增/修改/动作触发。
> - 不重复计算兼容路径，例如 `/api/templates` 和 `/api/sms/templates` 只算 1 个接口。
> - `GET /s/{shortCode}` 虽然会记录点击，但属于短链跳转，不计入新增/修改接口，单独备注。

## 1. 结论

| 类型 | 数量 | 说明 |
| --- | ---: | --- |
| 当前前端已实现路由视图 | 7 | 当前 `main.tsx` 中的 7 个 view |
| PRD 建议后台路由页面 | 28 | 不含弹窗、详情表单 |
| PRD 建议弹窗/详情/表单交互面 | 14 | 禁止使用抽屉，统一使用 Modal 或独立详情页 |
| 当前已实现新增/修改接口 | 9 | 接口文档中已落地的写接口 |
| 当前前端已接入新增/修改接口 | 8 | 当前页面未单独接入 `/api/sms/send-test-code` |
| PRD 规划新增/修改接口 | 34 | 账号、用户、黑名单、系统配置、审批、导出等后续接口；状态变更按一个 `/status` 接口统计 |
| V1 总新增/修改接口 | 43 | 已实现 9 + 规划 34 |

## 5. 当前已实现新增/修改接口

已实现写接口共 9 个；当前前端实际接入 8 个。

| 序号 | 接口 | 动作 | 当前前端是否接入 |
| ---: | --- | --- | --- |
| 1 | `POST /api/templates` | 创建模板 | 是 |
| 2 | `POST /api/templates/{templateId}/status` | 启用/停用模板 | 是 |
| 3 | `POST /api/rules` | 创建规则 | 是 |
| 4 | `POST /api/rules/{ruleId}/status` | 启用/停用规则 | 是 |
| 5 | `POST /api/manual-send` | 手动发送短信 | 是 |
| 6 | `POST /api/sms/send-test-code` | 发送测试验证码 | 否 |
| 7 | `POST /api/events` | 上报业务事件 | 是 |
| 8 | `POST /api/tasks/run-due` | 执行到期任务 | 是 |
| 9 | `POST /api/sms/provider/callback` | 接收/模拟回执 | 是 |

备注：`GET /s/{shortCode}` 会记录点击并跳转，但接口语义是短链访问，不计入新增/修改接口。
