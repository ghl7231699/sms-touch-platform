## 推荐阅读顺序

1. 先读 [项目摘要](project-summary.md)，建立全局理解。
2. 再读 PRD，确认业务目标和 V1 范围。
3. 按实现职责阅读规则中心、事件定义和技术设计。
4. 进入开发或测试前，对照技术设计中的接口、数据表和验收标准。

## V1 范围概览

V1 目标是形成完整链路：业务事件 -> 规则匹配 -> 生成短信任务 -> 调用短信服务商 -> 接收回执 -> 用户点击短链 -> 统计效果。

V1 支持短信模板管理、手动发送、自动规则触发、短链追踪、回执状态管理和效果统计分析。
