# 产品、前端、后端后续功能分工

本文档按产品、前端、后端三个角色整理下一阶段需要补充的功能。优先级分为：

- P0：上线核心能力，不做会阻塞真实业务联动。
- P1：商业化产品必备能力，影响运营效率和安全治理。
- P2：增长、规模化和生产增强能力。

## 总体判断

当前平台已经具备短信触达的主链路：

```text
模板 -> 规则 -> 事件 -> 任务 -> 条件判断 -> 发送 -> 回执 -> 短链 -> 统计
```

下一阶段的重点不应继续堆测试功能，而是补齐三条线：

1. 线上系统接入：让用户中心、订单系统、会员系统能安全上报事件和查询状态。
2. 运营规则生产化：让规则配置更可靠、可解释、可审计、可回滚。
3. 商业化后台治理：权限、审计、黑名单、频控、真实短信上线准备。

## 产品侧

### P0：定义线上事件联动规则

产品需要把“哪些业务行为触发短信”定义清楚。

| 功能 | 说明 | 交付物 |
| --- | --- | --- |
| 事件清单 | 明确 V1/V2 支持哪些业务事件 | 事件字典 |
| 事件来源 | 明确事件由哪个系统上报 | 来源系统清单 |
| payload 字段 | 定义每类事件必须携带哪些字段 | 事件字段说明 |
| 幂等规则 | 定义 `eventId` 生成规则，避免重复触发 | eventId 规范 |
| 异常处理 | 重复事件、缺手机号、缺 userId 时如何处理 | 错误处理规则 |

建议优先补齐这些事件：

| 事件 | 来源系统 | 用途 |
| --- | --- | --- |
| `user_register` | 用户中心 | 注册后未购买会员提醒 |
| `membership_purchased` | 会员/订单系统 | 转化归因、取消待发送任务 |
| `membership_expired` | 会员中心 | 会员过期召回 |
| `order_completed` | 订单系统 | 售后回访 |
| `campaign_start` | 活动中心 | 活动通知 |

### P0：定义核心业务规则模板

产品需要把运营常用规则沉淀成模板，避免每次从零配置。

| 规则模板 | 触发事件 | 条件 | 延迟 | 动作 |
| --- | --- | --- | --- | --- |
| 注册未购买会员 | `user_register` | 未购买会员 | 24 小时 | 发送会员转化短信 |
| 会员过期召回 | `membership_expired` | 仍未续费 | 3 天 | 发送续费短信 |
| 活动开始通知 | `campaign_start` | 用户订阅活动提醒 | 1 小时前 | 发送活动短信 |
| 订单完成回访 | `order_completed` | 未回访 | 7 天 | 发送回访短信 |

每个规则模板需要定义：

- 业务目标
- 触发事件
- 条件类型
- 延迟时间
- 短信模板
- 短链落地页
- 成功指标
- 退出条件

### P0：定义发送限制与防骚扰策略

短信触达天然敏感，产品必须先定义边界。

| 策略 | 建议 |
| --- | --- |
| 单手机号日频控 | 同手机号每天最多 1-2 条营销短信 |
| 单手机号周频控 | 同手机号每周最多 3-5 条营销短信 |
| 同场景冷却 | 同一场景 7 天内不重复触达 |
| 黑名单 | 投诉、退订、内部屏蔽号码不触达 |
| 已转化跳过 | 已购买会员、已续费、已完成目标的任务跳过 |
| 安静时段 | 夜间不发送，顺延到次日可发送时段 |

### P1：定义运营后台权限和流程

| 功能 | 产品决策 |
| --- | --- |
| 角色 | 管理员、运营、只读 |
| 模板权限 | 谁能创建、编辑、启停 |
| 规则权限 | 谁能创建、编辑、启停 |
| 发送权限 | 谁能手动发送真实短信 |
| 审批 | 是否需要规则上线审批 |
| 操作审计 | 哪些操作必须记录 |

### P1：定义统计指标

| 指标 | 说明 |
| --- | --- |
| 任务数 | 生成了多少触达任务 |
| 发送数 | 实际调用短信服务商的数量 |
| 成功数 | 服务商提交成功或回执成功 |
| 跳过数 | 条件不满足、已购买、黑名单、频控导致跳过 |
| 点击数 | 短链点击次数 |
| 点击率 | 点击人数 / 成功发送人数 |
| 转化数 | 点击或发送后购买会员的人数 |
| 转化率 | 转化人数 / 成功发送人数 |
| 退订/投诉 | 用户负反馈 |

### P2：定义增长玩法

| 方向 | 功能 |
| --- | --- |
| 人群运营 | 用户分群、人群包导入 |
| A/B 实验 | 不同文案、延迟时间、落地页对比 |
| 营销旅程 | 多步骤触达，不只单条短信 |
| 转化归因 | 购买、续费、回访结果回流 |
| 多通道 | 短信、站内信、企微、Push |

## 前端侧

### P0：线上事件联动可视化

前端需要让运营和研发能看到线上系统接入是否正常。

| 页面/模块 | 功能 |
| --- | --- |
| 事件来源管理 | 来源系统列表、状态、密钥展示/重置、最近调用时间 |
| 事件流水 | 事件状态、来源系统、eventId、失败原因、payload 摘要 |
| 事件详情 | 原始 payload、匹配规则、生成任务、处理日志 |
| 接入说明 | 展示上报 URL、签名规则、示例代码 |

### P0：规则中心升级

当前规则表单已经可配置基础条件，但还需要产品级体验。

| 功能 | 说明 |
| --- | --- |
| 结构化条件表单 | 不让用户手填 `conditionType` 字符串 |
| 条件类型组件 | 未购买会员、会员过期、活动开始前、订单完成后 |
| 条件预览 | 用自然语言展示规则：“注册后 24 小时仍未购买会员则发送” |
| 规则测试 | 输入模拟事件，预览会生成哪些任务 |
| 规则影响范围 | 预估可能触达人数、命中事件数 |
| 规则启停确认 | 对真实发送规则做二次确认 |

### P0：任务详情与跳过原因

条件判断上线后，运营必须看得懂为什么发或没发。

| 功能 | 说明 |
| --- | --- |
| 任务详情页 | 展示事件、规则、模板、条件、发送记录 |
| 条件结果 | `passed/skipped/error` 可视化 |
| 跳过原因 | 已购买会员、黑名单、频控、规则停用 |
| 重试操作 | 失败任务可手动重试 |
| 取消操作 | 待发送任务可人工取消 |

### P1：权限与审计界面

| 页面/模块 | 功能 |
| --- | --- |
| 登录页 | 管理后台登录 |
| 用户管理 | 管理员、运营、只读 |
| 操作日志 | 查看谁改了模板、规则、worker、发送 |
| 审批记录 | 如果后续加审批，需要审批流页面 |

### P1：统计分析升级

| 功能 | 说明 |
| --- | --- |
| 按规则统计 | 每条规则任务、发送、点击、转化 |
| 按模板统计 | 不同短信模板效果 |
| 按场景统计 | 注册转化、会员召回、活动通知 |
| 趋势图 | 日发送量、点击率、转化率 |
| 漏斗图 | 任务 -> 发送 -> 点击 -> 转化 |

### P2：运营效率功能

| 功能 | 说明 |
| --- | --- |
| 批量导入 | CSV 上传手机号或用户 ID |
| 模板变量预览 | 输入变量后预览短信内容 |
| 规则复制 | 复制已有规则快速创建 |
| 草稿/发布 | 规则编辑后先保存草稿，再发布 |
| A/B 配置 | 多模板、多延迟时间测试 |

## 后端侧

### P0：事件接入鉴权

线上系统接入前，必须给事件 API 加安全校验。

| 功能 | 说明 |
| --- | --- |
| `event_source` 表 | 保存来源系统 appId、secret、状态 |
| HMAC 签名 | 校验 `X-App-Id`、`X-Timestamp`、`X-Nonce`、`X-Signature` |
| 时间窗校验 | 例如只接受 5 分钟内请求 |
| nonce 防重放 | 同 appId + nonce 只能使用一次 |
| 接入日志 | 保存成功/失败调用记录 |

建议请求头：

```text
X-App-Id: user-center
X-Timestamp: 1780700000000
X-Nonce: random-string
X-Signature: hmac-sha256
```

### P0：条件评估器扩展

当前已支持 `not_purchased_membership`，后端需要继续扩展。

| 条件 | 数据来源 | 行为 |
| --- | --- | --- |
| `not_purchased_membership` | 会员/订单系统 | 已购买则跳过 |
| `membership_expired_after_days` | 会员系统 | 未续费则发送 |
| `before_campaign_start` | 活动系统 | 活动未开始且用户订阅则发送 |
| `after_order_completed` | 订单系统 | 订单完成且未回访则发送 |
| `not_in_blacklist` | 黑名单表 | 黑名单跳过 |
| `frequency_allowed` | 发送日志/频控表 | 超频跳过 |

### P0：任务取消与状态完善

| 功能 | 说明 |
| --- | --- |
| `skipped` | 条件不满足 |
| `cancelled` | 人工取消、规则停用、用户已转化 |
| `condition_error` | 外部条件服务查询失败 |
| 任务取消 API | 支持取消待发送任务 |
| 规则停用联动 | 规则停用后可取消未执行任务 |

### P1：线上服务适配器

| 适配器 | 功能 |
| --- | --- |
| 用户中心 | 查询用户基础信息、手机号、状态 |
| 会员系统 | 查询会员状态、过期时间、续费记录 |
| 订单系统 | 查询是否购买会员商品、订单状态 |
| 活动系统 | 查询活动状态、开始时间、订阅状态 |
| 黑名单服务 | 查询手机号是否允许触达 |

后端需要统一：

- timeout
- retry
- circuit breaker
- 鉴权 token
- 错误码映射
- 查询日志

### P1：权限、审计、登录

| 功能 | 说明 |
| --- | --- |
| 管理员账号 | 登录后台 |
| session/token | API 鉴权 |
| RBAC | 管理员、运营、只读 |
| `operation_audit_log` | 记录所有后台敏感操作 |
| API 权限中间件 | 保护模板、规则、发送、worker 操作 |

### P1：黑名单与频控

| 功能 | 说明 |
| --- | --- |
| `sms_blacklist` | 手机号、来源、原因、创建时间 |
| 退订接口 | 用户退订后加入黑名单 |
| 发送前校验 | 任务执行前统一检查 |
| 日/周频控 | 按手机号、场景、规则限制发送次数 |
| 安静时段 | 夜间任务顺延 |

### P2：正式短信与多服务商

| 功能 | 说明 |
| --- | --- |
| 正式短信 Provider | 接入正式营销短信 API |
| 服务商路由 | 按场景、成功率、费用选择服务商 |
| 回执映射 | 不同服务商回执统一为内部状态 |
| 失败降级 | 主服务商失败切备用服务商 |
| 费用统计 | 按服务商、规则、场景统计费用 |

### P2：转化归因

| 功能 | 说明 |
| --- | --- |
| `conversion_event` | 购买、续费、回访完成等转化事件 |
| 短链参数 | 携带 taskId、ruleId、userId |
| 归因窗口 | 例如点击后 7 天内购买算转化 |
| 漏斗统计 | 发送、点击、购买全链路 |

## 跨角色协作顺序

### 第一轮：线上事件安全接入

1. 产品定义来源系统、事件字段、失败处理。
2. 后端实现 `event_source`、签名校验、接入日志。
3. 前端实现事件来源管理和接入说明页面。
4. 联调用户中心上报 `user_register`。

### 第二轮：注册未购买会员完整闭环

1. 产品确认“未购买会员”的业务口径。
2. 后端完善会员系统适配器。
3. 前端完善规则条件表单和任务详情。
4. 联调用户注册、会员购买、任务跳过、短信发送。

### 第三轮：运营安全治理

1. 产品定义角色权限和审计范围。
2. 后端实现登录、RBAC、审计日志。
3. 前端实现登录、用户管理、操作日志。
4. 对真实短信发送增加二次确认和审计。

### 第四轮：增长和生产化

1. 产品定义分群、批量触达、转化归因口径。
2. 后端实现人群包、正式短信、多服务商、归因事件。
3. 前端实现批量导入、统计看板、漏斗分析。

## 近期建议排期

| 周期 | 产品 | 前端 | 后端 |
| --- | --- | --- | --- |
| 第 1 周 | 事件字段、来源系统、注册未购口径 | 事件来源管理原型、规则条件表单升级 | 事件鉴权、接入日志、event_source |
| 第 2 周 | 权限角色、频控策略 | 任务详情、规则测试、接入说明 | 会员/订单适配器、任务取消、频控基础 |
| 第 3 周 | 统计口径、转化定义 | 统计升级、操作日志页面 | 登录/RBAC/审计日志 |
| 第 4 周 | 正式短信上线方案 | 批量导入、人群包初版 | 正式短信 Provider、黑名单、退订 |

## 最小可上线版本建议

最小可上线版本至少应包含：

- 事件接入鉴权
- 事件来源管理
- `not_purchased_membership` 查询线上会员/订单系统
- 任务发送前二次校验
- 黑名单
- 频控
- 登录和操作审计
- 真实短信发送二次确认

这些完成后，才建议从测试短信链路进入小范围真实营销短信灰度。
